Tenets

  • Use semantic markup
  • Design for all screens; customize for larger screens.
  • CSS is not the unit of component re-use.
  • A limited set of sizes can solve most design problems.

See “But Why?” for justification and background on these tenets.

Using Media Queries & Breakpoints

Melange's classes apply to all screens, which implies any design using those classes affects mobile devices. If a design cannot work on both small and large screens, you can use suffixed classes to target largest screens, thus achieving a mobile-first design.

Suppose we wish the following component to appear like so on desktop:

Emperor of the Known Universe
arrakis.planet

and like so on mobile:

Paul Muad'Dib

Emperor of the Known Universe

Ideally, this can be achieved with the same markup. Start by making it work for mobile screens:

<div class="ba br-3"> <div class="flex flex-column items-center"> <img src="/muad-dib.png" class="mt-2 w-5 h-5 br-3" /> <h4 class="f-4 mb-2 mt-1">Paul Muad'Dib</h4> </div> <div class="bg-black white pa-2 br-bottom-3"> <h5 class="tc f-3 i mv-2">Emperor of the Known Universe</h5> <a class="db tc blue-light" href="https://arrakis.planet">arrakis.planet</a> </div> </div>

To change the design for larger screens, we must: 1) arrange the image and name horizontally, with appropriate padding for that layout, and 2) align the title and link left. To do this, you can use Melange's classes suffixed with -ns for “not small”, meaning any screen whose width is not considered small (thus, mobile).

<div class="ba br-3"> <div class="flex flex-column items-center flex-row-ns mv-3-ns"> <img src="/muad-dib.png" class="mt-2 w-5 h-5 br-3 mt-0-ns ml-2-ns" /> <h4 class="f-4 mb-2 mt-1">Paul Muad'Dib</h4> </div> <div class="bg-black white pa-2 br-bottom-3"> <h5 class="tc f-3 i mv-2 tl-ns">Emperor of the Known Universe</h5> <a class="db tc blue-light tl-ns" href="https://arrakis.planet">arrakis.planet</a> </div> </div>

Re-using Duplicate Classes

You can certainly imagine that on any reasonable app, you will be repeating strings of classes and this can lead to issues with managing change, or simply re-using commonly-grouped styles. For example, you may standardize on buttons with a horizontal padding of 3 and a vertical padding of 2 and centered text, with a border radius of 4.

If you are building a site by hand-editing HTML and CSS, abstracting these classes would very well be difficult to achieve. One way to use CSS is by creating a name for such styling and re-using that name everywhere. But most people aren't writing HTML files, they are building web apps, and web apps invariable have a templating system.

Melange assumes you can use that templating system to abstract any combination of markup and classes you need to re-use. Consider button styling with a few options: a small or a large button, and a primary and a secondary button:

Your templating system, including front-end frameworks like React, should be able to provide a way to generate this markup based on parameters or properties. For example:

function Button({size,type,label}) { const dimensions = size == "big" ? "ph-4 pv-3 br-4 f-4" : "ph-3 pv-2 br-3 f-3" const colors = type == "primary" ? "bg-blue-darkest white" : "bg-white blue-darkest" const cssClasses = `${dimensions} ${colors} ba tc` return ( <button className={cssClasses}> { label } </button> ) }

Or, in Rails:

def styled_button(size, type, label) dimensions = size == "big" ? "ph-4 pv-3 br-4 f-4" : "ph-3 pv-2 br-3 f-3" colors = type == "primary" ? "bg-blue-darkest white" : "bg-white blue-darkest" const cssClasses = `${dimensions} ${colors} ba tc` content_tag(:button, class: "#{dimensions} #{colors} ba tc") do label end end

These are just examples. You can do it however you want. And, because Melange is just a CSS file with variables, you could certainly create your own CSS if you like:

.button { border: solid; text-align: center; } .button.primary { color: #fff; background-color: var(--mg-bluedarkest); } .button.primary { color: var(--mg-bluedarkest); background-color: #fff; } .button.big { border-radius: var(--mg-bw4); padding: var(--mg-sp3) var(--mg-sp4); font-size: var(--mg-fs4); } .button.small { border-radius: var(--mg-bw3); padding: var(--mg-sp2) var(--mg-sp3); font-size: var(--mg-fs3); }
