Get Started

Getting Started Guide Reference Source Code Gallery But Why?

MelangeCSS can be installed in a few ways, depending on your needs. To get started now, choose which method:

  • CDN (quickest to get started)
  • Download (most stable)
  • NPM (most compatible with web frameworks)

MelangeCSS also ships with different variants, depending on your needs:

  • Do you need Dark Mode support?
    • Yes. Do you need any breakpoints?
      • Yes, all four.
        →

        You want melange.css.

        Most flexible.

      • Only mobile and desktop.
        →

        You want melange-breakpoint_ns_only.css.

        Saves space.

      • No.
        →

        You want melange-no_breakpoints.css.

        Good for single screen size designs.

    • No. Do you need any breakpoints?
      • Yes, all four.
        →

        You want melange-no_dark_mode.css.

        Saves some space when dark mode isn't needed.

      • Only mobile and desktop.
        →

        You want melange-no_dark_mode-breakpoint_ns_only.css.

        Saves more space, less flexible.

      • No.
        →

        You want melange-no_dark_mode-no_breakpoints.css.

        Good for simple internal apps aimed at desktop screens.

Dark Mode; All Breakpoints

  • CDN

    You can bring MelangeCSS into any webpage using the unpkg.com CDN provided for all NPM packages. Place this somewhere in your <head> section:

    <link rel="stylesheet" href="https://unpkg.com/melange-css@1.0.0-beta2/melange.min.css"/>

    Using the CDN, you can get started right awway. For production use, we recommend you avoid this method as it creates a runtime dependency on unpkg.com, which is not desirable.

  • Download

    To avoid a runtime dependency on unpkg.com, you can download melange.css with a command-line tool like curl:

    curl https://unpkg.com/melange-css@1.0.0-beta2/melange.min.css > \ your/project/css/melange.css

    This method is most stable, because you have a copy of MelangeCSS in your app and that copy won't be changed in any way without you explicitly changing it. The downside is that you don't have a way to easily update it when there are changes. To do that, use NPM.

  • NPM

    MelangeCSS is available as a Node Module, which you can install thusly:

    npm install --save melange-css

    The CSS will be in node_modules/melange-css/melange.css. How you get that into your app highly depends on how your app is built. If you have a build system that supports @import, you can bring MelangeCSS in by adding this to a .css file your build system processes:

    @import "melange-css/melange.css";

    If you use this method, you are encouraged to monitor the releases. MelangeCSS uses Semantic Versioning, so you should not experience breaking changes outside of a major release.
