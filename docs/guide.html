Tenets

  • Use semantic markup
  • Design for all screens; customize for larger screens.
  • CSS is not the unit of component re-use.
  • A limited set of sizes can solve most design problems.

See “But Why?” for justification and background on these tenets.

Guides

  • Step-by-Step Overview

    Learn how to style a component one step at a time, by using Melagne's classes.

  • Class Naming Conventions

    Melange's class names are terse, but they following straightforward conventions.

  • Using Media Queries & Breakpoints

    Melange allows you to use the same markup, but apply different styles for different media queries to make sure your app looks good on mobile and desktop.

  • Re-using Duplicate Classes

    Because you don't write CSS when using Melange, you'll use your web app's templating system for re-use.

  • Customzing Melange

    Melange can be customized to your needs, from changing fonts and colors, to creating an entirely different grid and scale.
